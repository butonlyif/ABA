# ABA 智能助手 — 操作手册

> 版本：v1.4 | 更新日期：2026-06-05  
> 适用产品：ABA 智能助手 + 人生教练 + 专家后台

## 一、用户使用指南

### 1.1 注册与登录

1. 浏览器打开 `http://服务器IP:8501`
2. 首次使用点击「注册」，填写用户名和密码
3. 登录后即进入 ABA 助手主页
4. **一个账号通用全部模块**——注册一次，ABA 助手、人生教练、后台管理均可使用

### 1.2 添加孩子档案

1. 侧边栏 → 「孩子档案」
2. 点击「添加孩子」，填写姓名、出生日期、诊断信息等
3. 支持添加多个孩子，每个孩子独立管理

### 1.3 使用流程

推荐的使用流程：

```
能力评估 → 查看结果 → 生成训练任务 → 开始训练 → 记录数据 → 查看进展
```

#### 步骤 1：能力评估

1. 侧边栏 → 「评估」
2. 选择要评估的孩子
3. 按阶段回答 39 道递进题（分 5 个阶段，每阶段 6-9 题）
4. 每题根据日常观察选择"是"或"否"
5. 完成后点击「完成评估，查看结果」

#### 步骤 2：自动生成训练任务

1. 在评估结果页查看各领域得分
2. 点击「一键生成全部训练任务」
3. 系统自动为孩子的薄弱领域创建训练任务
4. 前往「任务清单」查看

#### 步骤 3：执行训练

1. 任务清单中点击「开始训练」
2. 进入训练记录页，系统已自动选中对应技能
3. 按试次录入训练数据（正确/错误/辅助/无反应）
4. 完成后保存训练记录

### 1.4 主要页面说明

| 页面 | 功能 |
|------|------|
| **AI 对话** | 向 AI 提问 ABA 相关问题，基于知识库回答 |
| **孩子档案** | 管理孩子信息，查看进展记录 |
| **评估** | 39 题 5 阶段能力评估，自动生成训练任务 |
| **任务清单** | 查看和管理所有训练任务 |
| **训练记录** | ABA 训练数据记录，9 领域 210 技能可选 |
| **图片卡片** | 127 类别 6569 张教学图片卡，浏览/配对 |
| **进展看板** | 训练数据可视化 |
| **报告** | AI 生成训练进展报告 |
| **进入人生教练** | 跳转到家长心理成长教练（8503） |

### 1.5 人生教练使用

1. 从 ABA 助手侧边栏点击「进入人生教练」自动跳转（免登录）
2. 初次进入会引导设置成长阶段
3. 核心功能：教练对话 / 成长项目 / 情绪追踪 / 反思日记 / 知识库阅读
4. 点击「返回 ABA 助手」跳回主应用

## 二、评估体系说明

### 2.1 评估概述

| 项目 | 说明 |
|------|------|
| **总题数** | 39 题，均为"是/否"二选一 |
| **阶段数** | 5 个递进阶段 |
| **覆盖范围** | 9 个核心领域 × 210 个训练技能（100% 覆盖） |
| **答题方式** | 凭日常观察回答，无需现场测试 |

### 2.2 5 个阶段

| 阶段 | 题数 | 评估内容 | 对应等级 |
|------|------|----------|---------|
| 第 1 阶段 | 9 题 | 所有领域基本能力摸底 | Level 1 |
| 第 2 阶段 | 9 题 | 对基础较好领域向上推进 | Level 2 |
| 第 3 阶段 | 9 题 | 进一步提升能力层级 | Level 3 |
| 第 4 阶段 | 6 题 | 挑战较复杂技能 | Level 4 |
| 第 5 阶段 | 6 题 | 对标典型发育同龄水平 | Level 5 |

### 2.3 评分逻辑

- **递进评分**：每领域从低到高逐题判断
- 答"是" → 该层级技能视为已掌握，继续下一题
- 答"否" → 该层级及更高层级的全部技能加入推荐训练清单，该领域不再推进
- **优先级排序**：参与→模仿→语言→游戏→社交→视觉空间→情绪→自理→学业前

### 2.4 评估等级

| 等级 | 描述 | 训练建议 |
|------|------|---------|
| Level 1 | 基础阶段 | 从参与、模仿的最基础项目开始 |
| Level 2 | 中级阶段 | 推进语言、游戏、社交训练 |
| Level 3 | 进阶阶段 | 聚焦对话、合作游戏、情绪调节 |
| Level 4 | 高级阶段 | 挑战复杂社交、抽象语言、学前准备 |
| Level 5 | 专家阶段 | 精炼高阶技能，巩固薄弱项 |

## 三、部署运维指南

### 3.1 前置要求

- 一台 Ubuntu 22.04 服务器（建议 2 核 4G+）
- Docker + Docker Compose V2
- 腾讯云安全组开放 TCP 8501、8503

### 3.2 服务器初始化

```bash
# 装 Docker
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER
newgrp docker

# 验证
docker --version && docker compose version
```

### 3.3 部署

```bash
# 在本地项目根目录执行
bash deploy/deploy.sh                  # 全量部署
bash deploy/deploy.sh main             # 仅主应用
bash deploy/deploy.sh admin            # 仅后台
```

部署完成后访问：
- 主应用：`http://服务器IP:8501`
- 人生教练：`http://服务器IP:8503`

### 3.4 访问后台

后台不暴露公网，通过 SSH 隧道访问：

```bash
# 本地新开终端
bash deploy/tunnel.sh
# 浏览器打开 http://127.0.0.1:8502
```

### 3.5 图片卡预渲染

部署后自动运行预渲染。手动执行：

```bash
docker exec aba-assistant python utils/prerender_flashcards.py
```

缓存位置：`data/flashcard_cache/`（127 类别完成后约 3.6GB）。

### 3.6 日常运维命令

```bash
# 查看状态
docker compose -f deploy/docker-compose.yml ps

# 查看日志
docker compose -f deploy/docker-compose.yml logs -f --tail=200

# 重启
docker compose -f deploy/docker-compose.yml restart

# 备份数据
tar -czf backup-$(date +%F).tar.gz deploy/data/

# 完全重置
docker compose down && rm -rf deploy/data && docker compose up -d
```

### 3.7 配置 HTTPS

等有域名后：

1. 将域名 A 记录指向服务器 IP
2. 装 Nginx + certbot，配置反向代理（需开启 WebSocket 支持）
3. `sudo certbot --nginx -d your.domain.com` 签发证书
4. 安全组关闭 8501/8503，仅暴露 80/443

### 3.8 故障排查

| 现象 | 排查 |
|------|------|
| 浏览器打不开 | `docker compose ps` 看容器状态；安全组端口是否开放 |
| 进入人生教练失败 | 安全组 8503 是否开放；`LIFE_COACH_URL` 配置是否正确 |
| 教练要重新登录 | 两个 service 的 `COACH_SSO_SECRET` 不一致 |
| 登录后报 API 错误 | `.env` 中 API Key 是否填写 |
| 容器 unhealthy | `docker compose logs --tail=200` 查看启动日志 |
| 后台索引报错 | 删除 `deploy/data/users/admin_vectors/` 再重建 |

## 四、知识库构建指南

### 4.1 添加新知识库文档

1. 在 `docs/知识库/` 目录下新建 `.md` 文件
2. 文件命名格式：`XX_文档名称.md`（如 `09_新主题.md`）
3. 使用清晰的 Markdown 层级标题

```markdown
# 文档标题

## 第一节

正文内容...

## 第二节

- 要点 1
- 要点 2
```

### 4.2 文档编写规范

- 每个 `.md` 文件控制在 2000-5000 行为佳
- 用 `##` 和 `###` 做层级切分，ChromaDB 会按段落自动分块
- 引用的专业内容标注来源
- 避免大段纯文本，优先使用列表和表格

### 4.3 部署后索引更新

修改知识库文档后，重新部署即可自动重建索引：

```bash
bash deploy/deploy.sh
```

Dockerfile 中的 `RUN python ingest_kb.py` 会在构建时自动重新索引全部文档。

## 五、常见问题 FAQ

### Q: 评估要全部答完吗？
A: 建议答完 5 个阶段以获得完整评估和任务清单。也可以只答前几阶段，系统会根据已答题目生成推荐。

### Q: 评估结果可以重新做吗？
A: 可以。在评估结果页点击「重新评估」即可。

### Q: 训练任务太多怎么办？
A: 系统按领域优先级排序（参与→模仿→语言→...），建议从上往下依次完成，优先打好参与和模仿的基础。

### Q: 如何备份数据？
A: `tar -czf backup-$(date +%F).tar.gz deploy/data/`。`flashcard_cache/` 可重建，备份时加 `--exclude flashcard_cache` 跳过。

### Q: 换服务器怎么迁移？
A: 备份 `deploy/data/` 目录到新服务器，重新 `deploy.sh` 部署，恢复数据即可。

### Q: 图片卡片打开很慢？
A: 等待预渲染完成（约 30-40 分钟），之后所有卡片秒开。可查看进度：`docker exec aba-assistant ls data/flashcard_cache/ | wc -l`（目标 127）。
