# ABA 平台现代化实施说明

新平台与原有 Streamlit 应用并行存在。旧系统没有被删除，迁移完成并
通过数据对账前不得停止或覆盖旧数据。

## 本地启动

后端：

```bash
cd apps/api
python -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/uvicorn app.main:app --reload
```

手机端：

```bash
npm install
npm run dev:web
```

访问 `http://localhost:5173`；API 文档位于
`http://localhost:8000/docs`。

## 数据迁移

先运行只读检查：

```bash
cd apps/api
python scripts/migrate_legacy.py --legacy-root ../../src/MVP_web/data
```

查看 `migration-report.json` 后，再在已经备份和配置好目标数据库的
环境中加入 `--apply`。迁移器使用稳定 ID 映射并可重复执行，绝不删除
源 SQLite 或 JSON 文件。

## 发布顺序

1. 修复旧系统回归测试并冻结关键行为。
2. 在预发布环境执行 Alembic 和迁移演练。
3. 让内部测试用户使用 PWA 完成整条训练流程。
4. 对账用户、孩子、任务、训练试次和报告。
5. 主产品切流；旧 Streamlit 保持只读。
6. 迁移教练与专家后台后再归档旧服务。

当前 API 已覆盖认证、孩子、评估、任务、训练、进展、报告和安全降级
问答。教练数据持久化、AI RAG 适配器、对象存储与后台 Worker 是下一
增量，现有接口边界不会因此改变。
